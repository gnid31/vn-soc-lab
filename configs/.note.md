# `configs/` — production configuration files

These files mirror what is deployed on the VPS / endpoint. All credentials are stubbed with placeholders:

| Placeholder | Real value lives in |
|---|---|
| `<ELASTIC_PASSWORD>` | `/home/namth/elastic-credentials.txt` on the VPS (chmod 600) |
| `<HOSTNAME>` | substituted at deploy time with `$env:COMPUTERNAME` (Windows) |

When applying on the target machine, `sed -i` the placeholder out before `systemctl restart`.
