## 1. Vì sao detect cái này

Hợp pháp: trình duyệt (chrome.exe, msedge.exe, firefox.exe) là nguồn duy nhất truy cập HTTP/HTTPS từ user-context. Các app khác có nhu cầu HTTPS thường là **system service** (Windows Update, OneDrive, Teams) — chạy từ `svchost.exe` hoặc binary đã ký Microsoft.

Bất thường: `powershell.exe`, `cmd.exe`, `wscript.exe`, `mshta.exe`, custom binary trong `%APPDATA%`/`%TEMP%` mà mở kết nối ra port 80/443 → cực kỳ khả năng cao là **beacon C2** hoặc **dropper download payload**.

MITRE T1071.001 (Web Protocols) là tactic Command-and-Control phổ biến nhất — Cobalt Strike default profile, Empire, Sliver, các webshell callback.

**Severity Medium** vì rule này có FP rate trung bình (Windows tự ship nhiều background process gọi HTTPS). Severity High dành cho R1/R4.

## 2. KQL query

```
event.code: "3" AND
winlog.provider_name: "Microsoft-Windows-Sysmon" AND
winlog.event_data.DestinationPort: ("80" OR "443" OR "8080" OR "8443") AND
NOT winlog.event_data.Image.keyword: (
  *\\chrome.exe OR
  *\\msedge.exe OR
  *\\firefox.exe OR
  *\\iexplore.exe OR
  *\\opera.exe OR
  *\\brave.exe OR
  *\\OneDrive.exe OR
  *\\Teams.exe OR
  *\\ms-teams.exe OR
  *\\svchost.exe OR
  *\\RuntimeBroker.exe OR
  *\\WindowsTerminal.exe OR
  *\\SearchApp.exe OR
  *\\backgroundTaskHost.exe OR
  *\\NisSrv.exe OR
  *\\MsMpEng.exe OR
  *\\smartscreen.exe OR
  *\\WUDFHost.exe OR
  *\\YourPhone.exe OR
  *\\WidgetService.exe OR
  *\\HxOutlook.exe OR
  *\\HxTsr.exe OR
  *\\ApplicationFrameHost.exe OR
  *\\CompPkgSrv.exe OR
  *\\agy.exe                  /* Antigravity AI agent — tuned 2026-06-25 */
)
```

**Giải thích:**

- `event.code: "3"` — Sysmon network connection.
- `DestinationPort: ("80" OR "443" OR ...)` — chỉ HTTP/HTTPS ports phổ biến. C2 thường disguise qua 443.
- `NOT Image.keyword: (...)` — exclude list browser + Windows native HTTPS processes. Dùng `.keyword` vì path có `\`.

**Exclude list không cover 100% legit traffic.** Sau Pha 4 sẽ refine theo FP rate. Trong môi trường doanh nghiệp thật, exclude list có thể dài hơn (theo software chuẩn của công ty).

## 3. Ví dụ event match

**Suspicious (sẽ trigger):**

```yaml
event.code: "3"
winlog.provider_name: "Microsoft-Windows-Sysmon"
winlog.event_data.Image          : "C:\\Windows\\System32\\WindowsPowerShell\\v1.0\\powershell.exe"
winlog.event_data.DestinationIp  : "203.0.113.42"
winlog.event_data.DestinationPort: "443"
winlog.event_data.DestinationHostname: "evil.example.com"
winlog.event_data.User           : "DESKTOP-L7FCMBQ\\ADMIN"
```

**Legit (sẽ KHÔNG trigger — đã exclude):**

```yaml
event.code: "3"
winlog.event_data.Image: "C:\\Program Files\\Microsoft\\Edge\\Application\\msedge.exe"
winlog.event_data.DestinationPort: "443"
```

## 4. False-positive thường gặp & cách lọc

| FP scenario | Đặc trưng | Khuyến nghị |
|---|---|---|
| **`agy.exe` (Antigravity AI agent)** — **đã observed 2026-06-25** | Image `*\\AppData\\Local\\agy\\bin\\agy.exe` → Google API IPs port 443 | **Pha 4: bổ sung `*\\agy.exe` vào exclude list** (chỉ nếu confirm tool đã được verify hash). Lab dev có Antigravity → 30/35 alert là từ tool này |
| Chocolatey / winget update | Image trong `%ProgramData%\chocolatey\bin\` hoặc `%LocalAppData%\Microsoft\WinGet\` | Bổ sung exclude pattern theo path |
| `curl.exe` / `wget.exe` admin sử dụng | Image = `*\\curl.exe`, `*\\wget.exe` | Giữ alert — admin nên dùng PowerShell `Invoke-WebRequest`. Nếu môi trường thực tế có legit, exclude theo IntegrityLevel=High + parent=cmd.exe |
| `git.exe` clone HTTPS | Image = `*\\git.exe` | Bổ sung exclude nếu repo HTTPS hợp pháp; giữ nếu môi trường không dùng |
| Custom in-house app | Image trong `C:\Program Files\<CompanyName>\` | Whitelist theo path doanh nghiệp |
| `notepad.exe` open URL từ Help | Hiếm — chỉ Windows 11 với Notepad có Bing | Chấp nhận |

**Bài học SOC quan trọng (2026-06-25):** Rule **chính xác về kỹ thuật** (binary trong `%APPDATA%` gọi HTTPS — đúng pattern beacon C2), nhưng FP do **"known-good tool có hành vi giống malware"**. Doanh nghiệp thực giải quyết bằng tuning theo hash + path đã verify, KHÔNG bằng cách bỏ rule. Pha 4 sẽ analyze top FP source rồi refine.

## 6. Smoke-test (an toàn — IWR từ PowerShell)

**Trên Win10, PowerShell as Administrator:**

```powershell
# Sinh 1 outbound HTTPS từ powershell.exe — KHÔNG phải browser → trigger R5
Invoke-WebRequest -Uri https://example.com/ -UseBasicParsing | Out-Null
```

PowerShell.exe sẽ mở TCP connection → port 443 → IP của example.com. Sysmon e3 record event. Rule R5 match vì `Image=*\powershell.exe` không nằm trong exclude list.

**Đợi ≤5 phút** → Kibana → Security → Alerts → thấy alert `[VN-SOC R5]`.

**Cleanup:** không cần — chỉ là HTTP request, không tạo file/registry persistent.

**Lưu ý FP test:** sau khi smoke-test, rule có thể fire alert vài lần khác do background process gọi HTTPS (Windows Update, Office check-in). Đó là tín hiệu refine exclude list cho Pha 4.
