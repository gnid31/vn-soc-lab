## 1. Vì sao detect cái này

Registry Run Key là cơ chế persistence **đơn giản & cũ nhất** của Windows. Bất kỳ value name nào được ghi vào:

```
HKLM\Software\Microsoft\Windows\CurrentVersion\Run
HKLM\Software\Microsoft\Windows\CurrentVersion\RunOnce
HKCU\Software\Microsoft\Windows\CurrentVersion\Run
HKCU\Software\Microsoft\Windows\CurrentVersion\RunOnce
HKLM\Software\Wow6432Node\Microsoft\Windows\CurrentVersion\Run         (32-bit on 64-bit OS)
HKLM\Software\Wow6432Node\Microsoft\Windows\CurrentVersion\RunOnce
```

→ sẽ được Windows chạy mỗi lần user logon / boot. Malware sử dụng vì:

- Không cần Administrator (HKCU Run keys ghi được với user-context).
- Không bị Defender alert mặc định (chỉ alert nếu value point tới malware đã biết).
- Persistence ngay cả khi user reboot.

**Severity Medium** vì có nhiều legit software cũng ghi Run key (xem §4 FP).

## 2. KQL query

```
event.code: "13" AND
winlog.provider_name: "Microsoft-Windows-Sysmon" AND
winlog.event_data.TargetObject.keyword: (
  *\\Microsoft\\Windows\\CurrentVersion\\Run\\* OR
  *\\Microsoft\\Windows\\CurrentVersion\\RunOnce\\*
)
```

**Giải thích:**

- `event.code: "13"` — Sysmon registry value set.
- `TargetObject.keyword: *...Run\\*` — `.keyword` bắt buộc (path có `\`). Pattern `Run\\*` (có trailing backslash + wildcard) đảm bảo match value-name child của key Run, KHÔNG match khoá tên giống `Run` ở chỗ khác (vd `Runtime`).
- Pattern bao gồm cả `Run` và `RunOnce` cho cả HKLM, HKCU, Wow6432Node (1 wildcard `*\\Microsoft\\Windows\\CurrentVersion\\Run\\*` đã cover hết các nhánh đầu).

**Đã loại bỏ pattern không cần thiết:** không thêm filter `Image` hay `User` ở phase đầu — muốn xem toàn bộ Run key writes, sau Pha 4 mới refine theo FP rate.

## 3. Ví dụ event match (sẽ có khi smoke-test §6)

```yaml
event.code: "13"
winlog.provider_name: "Microsoft-Windows-Sysmon"
winlog.event_data.EventType   : "SetValue"
winlog.event_data.TargetObject: "HKU\\S-1-5-21-...\\Software\\Microsoft\\Windows\\CurrentVersion\\Run\\VnSocTest"
winlog.event_data.Details     : "powershell.exe -NoP -W Hidden -Command Write-Host hi"
winlog.event_data.Image       : "C:\\Windows\\System32\\WindowsPowerShell\\v1.0\\powershell.exe"
winlog.event_data.User        : "DESKTOP-L7FCMBQ\\ADMIN"
```

## 4. False-positive thường gặp & cách lọc

| FP scenario | Đặc trưng | Khuyến nghị |
|---|---|---|
| Edge / Chrome / OneDrive auto-update | Image = signed Microsoft/Google binary | Whitelist `winlog.event_data.Image.keyword` theo path cụ thể |
| Windows installer / setup | Parent thường `msiexec.exe` | Whitelist `winlog.event_data.Image.keyword: *\\msiexec.exe` |
| Gaming clients (Steam, Epic) | Run key trỏ tới `Steam.exe`, `EpicGamesLauncher.exe` | Whitelist Details chứa `Steam` / `Epic` |
| Antivirus tự setup | Image trong `Program Files\<AV>` | Whitelist theo AV path |

**Ban đầu giữ rule rộng.** Trong môi trường doanh nghiệp VN, Run key write của user thường (ngoài Software installer) là dấu hiệu đáng nghi.

## 6. Smoke-test (an toàn — không cần tắt Defender)

**Trên Win10, PowerShell as Administrator (hoặc user thường):**

```powershell
# Sinh 1 Run key benign — point tới powershell.exe Write-Host
$RegPath = 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Run'
New-ItemProperty -Path $RegPath -Name 'VnSocR3Test' `
                 -Value 'powershell.exe -Command "Write-Host VN-SOC R3 test"' `
                 -PropertyType String -Force

# Verify đã ghi
Get-ItemProperty -Path $RegPath -Name 'VnSocR3Test'
```

**Đợi ≤5 phút** → Kibana → Security → Alerts (time range Last hour) → thấy alert `[VN-SOC R3] Registry Run Key Persistence`.

**Cleanup sau khi verify alert:**

```powershell
Remove-ItemProperty -Path 'HKCU:\Software\Microsoft\Windows\CurrentVersion\Run' `
                    -Name 'VnSocR3Test' -Force
```

Cleanup này KHÔNG trigger rule (Sysmon e14 = key delete, không phải e13 = value set).
