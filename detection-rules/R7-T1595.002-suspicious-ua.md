## 1. Vì sao detect cái này

Hacking tools mặc định để **User-Agent string đặc trưng** trong HTTP request — fingerprint dễ catch:

| Tool | Default UA pattern | Mục đích |
|---|---|---|
| sqlmap | `sqlmap/1.7.x` | SQL injection automation |
| Nikto | `Mozilla/5.00 (Nikto/2.x.x)` | Web server vuln scan |
| nmap NSE | `Nmap Scripting Engine` | Service version + script scan |
| masscan | `Masscan/1.x` | Mass port scan |
| dirb / gobuster / dirbuster | `Mozilla/5.0 zgrab/0.x` hoặc default | Directory bruteforce |
| Hydra | `Mozilla/4.0 (Hydra)` | Credential brute |
| Burp Suite Pro (default) | thường tự đổi nhưng Intruder mặc định có dấu vết | Manual pentest framework |

Attacker pro **có thể tự đổi UA** để bypass — nhưng đa số automated scan (script kiddie, mass scanning bot) dùng default UA. Catch UA fingerprint là **layer 1 detection**, bổ sung cho ET SCAN signature ở Suricata.

## 2. KQL query (final — đã verify)

```
user_agent.original: (
  *sqlmap* OR
  *nikto* OR
  *nmap* OR
  *masscan* OR
  *hydra* OR
  *nuclei* OR
  *gobuster* OR
  *dirbuster* OR
  *zgrab* OR
  *wpscan*
)
```

**Giải thích — 2 KQL bug đã debug trong Pha 6 (xem §2a):**

- Field `user_agent.original` — Logstash 8.x với ECS v8 mode auto-convert grok `agent` field → ECS path `user_agent.original`. Lúc viết spec dùng `agent` SAI vì `agent` field root bị Filebeat metadata overwrite.
- Wildcard KHÔNG có dấu `"`. Trong KQL `"*sqlmap*"` được hiểu là **literal string** với ký tự `*` thật — không match wildcard. Đúng là `*sqlmap*` (không quote).
- Wildcard `*xxx*` match substring case-insensitive (KQL default). Catch cả `sqlmap/1.7.2` và `(Nikto/2.5.0)`.

## 4. False-positive thường gặp & cách lọc

| FP scenario | Đặc trưng | Khuyến nghị |
|---|---|---|
| Authorized internal pentest team test | src_ip = team pentest, scheduled | Whitelist src_ip + thông báo trước qua change management |
| Bug bounty researcher hợp pháp | Đến từ IP công bố, có scope agreement | Whitelist IP của researcher đã enroll |
| Vendor automated security scanner (Qualys, Tenable) | Source IP cố định trong service contract | Whitelist theo IP + thời gian scan window |
| Sysadmin dùng `curl` với UA test | clientip nội bộ admin | Severity low — chấp nhận noise |

## 6. Smoke-test (an toàn — chỉ UA header, không exploit)

Từ Kali:

```bash
# Variety of suspicious UAs
curl -H "User-Agent: sqlmap/1.7.2" "http://192.168.154.165:8080/?id=1"
curl -H "User-Agent: Mozilla/5.00 (Nikto/2.5.0)" "http://192.168.154.165:8080/"
curl -H "User-Agent: Nmap Scripting Engine; https://nmap.org/book/nse.html" "http://192.168.154.165:8080/"
curl -H "User-Agent: gobuster/3.6" "http://192.168.154.165:8080/admin"
curl -H "User-Agent: WPScan v3.8.22" "http://192.168.154.165:8080/wp-login.php"
```

**Đợi ≤5 phút** → Kibana → Security → Alerts → R7 fire 5 alerts với từng UA pattern.
