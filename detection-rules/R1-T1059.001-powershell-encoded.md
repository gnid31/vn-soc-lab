| Rule ID | `vnsoc-r1-powershell-encoded` |
| MITRE Tactic | TA0002 — Execution |
| MITRE Technique | **T1059.001** Command and Scripting Interpreter: PowerShell |
| Severity | **High** |
| Risk score | 73 |
| Index pattern | `winlogbeat-*` |
| Rule type | Custom query (KQL) |
| Schedule | Every 5 minutes, look-back 10 minutes |
| Status | ⏳ Spec viết, chưa import Kibana |

## 1. Vì sao detect cái này

Tham số `-EncodedCommand` (hoặc viết tắt `-enc`, `-e`) của `powershell.exe` cho phép truyền payload Base64. Đây là kỹ thuật **#1 phổ biến nhất** để bypass detection theo command-line text, được sử dụng bởi:

- Emotet, TrickBot, Cobalt Strike beacons.
- Atomic Red Team T1059.001-1, T1059.001-2.
- Phần lớn macro-droppers từ phishing email Office.

Trong môi trường doanh nghiệp VN ngân hàng/fintech, **không có lý do hợp pháp** nào để user-level process dùng `-EncodedCommand`. Tools hệ thống (SCCM, GPO scripts) **có** dùng nhưng chạy dưới `SYSTEM` từ `svchost.exe` parent — phần này được loại trong false-positive filter ở §4.

## 2. KQL query

```
event.code: "1" AND
winlog.provider_name: "Microsoft-Windows-Sysmon" AND
(
  winlog.event_data.Image: (*\\powershell.exe OR *\\pwsh.exe OR *\\powershell_ise.exe)
) AND
(
  winlog.event_data.CommandLine: (
    *-EncodedCommand* OR
    *-encodedcommand* OR
    *-enc * OR
    * -e *JAB* OR
    * -e *SQB*
  )
)
```

**Giải thích từng phần:**

- `event.code: "1"` — Sysmon process creation. Dùng `event.code` (ECS keyword) thay vì `winlog.event_id` (text) để KQL nhanh hơn và không cần `.keyword` suffix.
- `winlog.provider_name: "Microsoft-Windows-Sysmon"` — defensive filter, tránh nhầm với Windows native 4688 (cũng có event.code=1 nếu có provider khác).
- `Image: *\\powershell.exe ...` — escape `\` đôi vì KQL parser. Bắt cả pwsh (PowerShell 7) và powershell_ise (ISE) không chỉ powershell.exe.
- `CommandLine: *-EncodedCommand*` / `*-enc *` — case-insensitive trong KQL mặc định, cover cả full và short form.
- `* -e *JAB*` và `* -e *SQB*` — `JAB` là 3 ký tự Base64 khởi đầu của `$` (`0x24`), `SQB` là `I` (`0x49`); 2 prefix này xuất hiện trong >95% encoded PowerShell vì payload bắt đầu bằng `$Invoke...` hoặc `IEX (`. Lưới chặn `-e` ngắn (1 ký tự) mà không có false-positive `-e` của lệnh khác.

## 4. False-positive thường gặp & cách lọc

| FP scenario | Đặc trưng | Khuyến nghị lọc |
|---|---|---|
| SCCM client task | Parent `svchost.exe`, user `SYSTEM` | Loại trừ `winlog.event_data.User: "NT AUTHORITY\\SYSTEM"` nếu chấp nhận giảm coverage cho privilege escalation |
| GPO logon scripts | Parent `gpscript.exe`/`userinit.exe`, user-context | Đánh dấu severity=Low thay vì exclude |
| AppLocker / Defender ATP scan | Parent `MsMpEng.exe`, hash đã sign Microsoft | Whitelist theo `winlog.event_data.IntegrityLevel: "System"` + `winlog.event_data.Hashes` chứa hash đã ký |

**Ban đầu giữ nguyên KQL, KHÔNG exclude gì.** Sau Pha 4 mới điều chỉnh theo FP rate thực tế trong lab.

## 5. Cấu hình rule trong Kibana — bước đầy đủ

1. Kibana → **Security → Rules → Detection rules → Create new rule**.
2. Chọn **Custom query**.
3. Source: **Custom Data View** → tạo data view mới `winlogbeat-*` nếu chưa có.
4. **Custom query**: paste KQL ở §2.
5. **Continue** → About rule:
   - **Name**: `[VN-SOC R1] PowerShell EncodedCommand Execution`
   - **Description**:
     ```
     Phát hiện process powershell.exe / pwsh.exe được tạo với tham số
     -EncodedCommand chứa payload Base64. Đây là kỹ thuật #1 trong
     T1059.001 PowerShell — bypass detection theo command-line text.
     Tham khảo: detection-rules/R1-T1059.001-powershell-encoded.md
     ```
   - **Severity**: High
   - **Risk score**: 73
   - **Tags**: `VN-SOC-Lab`, `T1059.001`, `PowerShell`
   - **MITRE ATT&CK threats**:
     - Tactic: `Execution (TA0002)`
     - Technique: `Command and Scripting Interpreter: PowerShell (T1059.001)`
6. **Continue** → Schedule:
   - Runs every: **5 minutes**
   - Additional look-back time: **5 minutes** (tổng look-back 10 phút)
7. **Continue** → Actions: bỏ qua.
8. **Create & enable**.
