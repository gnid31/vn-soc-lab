## 1. Vì sao detect cái này

Tham số `-EncodedCommand` (hoặc viết tắt `-enc`, `-e`) của `powershell.exe` cho phép truyền payload Base64. Đây là kỹ thuật **#1 phổ biến nhất** để bypass detection theo command-line text, được sử dụng bởi:

- Emotet, TrickBot, Cobalt Strike beacons.
- Atomic Red Team T1059.001-1, T1059.001-2.
- Phần lớn macro-droppers từ phishing email Office.

Trong môi trường doanh nghiệp VN ngân hàng/fintech, **không có lý do hợp pháp** nào để user-level process dùng `-EncodedCommand`. Tools hệ thống (SCCM, GPO scripts) **có** dùng nhưng chạy dưới `SYSTEM` từ `svchost.exe` parent — phần này được loại trong false-positive filter ở §4.

## 2. KQL query (final — đã verify smoke-test)

```
event.code: "1" AND
winlog.provider_name: "Microsoft-Windows-Sysmon" AND
winlog.event_data.Image.keyword: (
  *\\powershell.exe OR
  *\\pwsh.exe OR
  *\\powershell_ise.exe
) AND
winlog.event_data.CommandLine: (
  *EncodedCommand* OR
  *encodedcommand*
)
```

**Giải thích từng phần:**

- `event.code: "1"` — Sysmon process creation. Dùng `event.code` (ECS keyword) thay vì `winlog.event_id` (text) để KQL không cần `.keyword` suffix.
- `winlog.provider_name: "Microsoft-Windows-Sysmon"` — defensive filter, tránh nhầm với Windows native 4688 (cũng có event.code=1 nếu có provider khác).
- `Image.keyword: *\\powershell.exe ...` — dùng `.keyword` (xem §2a bài học). Backslash `\\` trong KQL = literal `\`. Bắt cả pwsh (PowerShell 7) và powershell_ise (ISE) không chỉ powershell.exe.
- `CommandLine: *EncodedCommand*` / `*encodedcommand*` — text field nhưng wildcard hoạt động vì `EncodedCommand` là 1 token nguyên (không có separator như `\` hay space). Case-insensitive trong KQL mặc định, 2 pattern viết hoa/thường để cover cả Windows mặc định và biến thể.

### 2a. Bài học quan trọng — KHI NÀO phải dùng `.keyword`

> **Phát hiện trong quá trình smoke-test 2026-06-25:** KQL R1 phiên bản đầu dùng `winlog.event_data.Image: *\\powershell.exe` trên text field → match **0 doc** mặc dù event đã vào ES. Phải đổi sang `Image.keyword` mới fire alert. Tham khảo report.md §6.7+.

**Quy tắc cho R3/R4/R5 và mọi rule tương lai:**

| Pattern KQL | Field type cần | Lý do |
|---|---|---|
| Chứa `\\` (đường dẫn Windows) | `.keyword` bắt buộc | Text analyzer tokenize `\` thành separator → wildcard không cross-token |
| Chứa space (vd `*-enc *`) | `.keyword` bắt buộc | Space cũng là separator |
| Substring trong 1 từ (vd `*EncodedCommand*`) | text OK | Token nguyên, wildcard substring trên 1 token vẫn hoạt động |
| Exact match từ đầu (vd `Image: powershell.exe`) | text OK | Token "powershell.exe" tồn tại |
| Aggregation / sort / threshold | `.keyword` bắt buộc | Text không fielddata |

**Trade-off của `.keyword`:** default `ignore_above=256` char → string > 256 sẽ không index → query miss. Sysmon `CommandLine` của attacker thật (Empire C2, Cobalt Strike beacon Base64) thường > 1000 char. Lab hiện tại chấp nhận, Pha 3+ sẽ migrate sang Winlogbeat-native index template (field `process.command_line` = `wildcard` type, không có ignore_above limit).

### 2b. Pattern đã loại bỏ so với phiên bản đầu

Phiên bản đầu của KQL có thêm 3 pattern bắt biến thể:

```
*-enc *      OR
* -e *JAB*  OR
* -e *SQB*
```

**Đã bỏ** vì cả 3 đều chứa **space** → fail trên text field (như giải thích ở §2a). Nếu muốn giữ, phải dùng `winlog.event_data.CommandLine.keyword` cho cả clause CommandLine — nhưng đụng ignore_above 256 trên attacker thật. Pha 4 sẽ đánh giá lại sau khi chạy Atomic Red Team với CommandLine dài.

## 4. False-positive thường gặp & cách lọc

| FP scenario | Đặc trưng | Khuyến nghị lọc |
|---|---|---|
| SCCM client task | Parent `svchost.exe`, user `SYSTEM` | Loại trừ `winlog.event_data.User: "NT AUTHORITY\\SYSTEM"` nếu chấp nhận giảm coverage cho privilege escalation |
| GPO logon scripts | Parent `gpscript.exe`/`userinit.exe`, user-context | Đánh dấu severity=Low thay vì exclude |
| AppLocker / Defender ATP scan | Parent `MsMpEng.exe`, hash đã sign Microsoft | Whitelist theo `winlog.event_data.IntegrityLevel: "System"` + `winlog.event_data.Hashes` chứa hash đã ký |

**Ban đầu giữ nguyên KQL, KHÔNG exclude gì.** Sau Pha 4 mới điều chỉnh theo FP rate thực tế trong lab.
